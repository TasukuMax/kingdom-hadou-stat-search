キングダム覇道の戦力検索では何ができますか？

戦力検索ページでは、武将の上位2ステータス、技能条件、連鎖率をもとに主将候補を絞り込めます。 育てたい武将を探す時や、主将適性を比較したい時に使いやすい構成です。

キングダム覇道のガチャシミュレーターはどんな内容ですか？

ガチャシミュレーターでは、シーズン3英傑登用の10連結果、SSR排出、天井、累計回数を確認できます。 本家寄せの結果画面と「もう10連する」導線を入れているので、結果確認も直感的です。

キングダム覇道のランキングは何が見られますか？

ランキングページでは、サーバー1・2・3の個別ランキングと、3サーバー統合の全体ランキングを見られます。 まずは戦闘力に絞って、上位1〜100人の名前と戦闘力を確認できる構成です。

このサイトは公式サイトですか？

このサイトはキングダム覇道ユーザー向けの非公式攻略ツールサイトです。 ゲーム・画像・名称などの権利は各権利者に帰属し、実際のゲーム内数値や仕様変更が入った場合は追って更新します。

S3 部隊戦力算出

シーズン3の動画と編成画像で見えた部隊戦力を元に、兵力・攻撃・防御・戦威・策略から 部隊戦力を見積もる簡易算出です。陣形バフやデバフも割合で直接かけられます。

算出の前提
  • シーズン3の動画・編成画面サンプルから、部隊戦力と表示ステータスの関係を回帰しています。
  • この算出は「最終的に画面へ出る兵力 / 攻撃 / 防御 / 戦威 / 策略」を使う簡易式です。
  • 兵力3000前後の通常部隊で精度が高く、編成比較や陣形比較の目安として使いやすい形に寄せています。

編成画面に出る兵力を入力します。通常は 3000 か 3090 / 3180 / 3300 付近です。

陣形や技能で兵力が伸びる時は、ここへ割合で入れます。減少はマイナスです。

武将戦力 仮説比較

提供画像15件と公開情報を突き合わせて、武将1体の戦力が何で決まりやすいかを複数仮説で比べる研究用パネルです。 うち5件は同時刻で将星・練達まで追えているため、単なる近似だけでなく「表示ステータスがどう分解されているか」も検証しています。

比較の前提
  • GameWith の「戦力は最大兵力と各ステータスから算出」を出発点に、能力画面で見える値だけでどこまで説明できるかを見ています。
  • 仮説は 4戦闘ステのみ / Lv追加 / 天賦追加 / 魅力追加 の順に増やし、LOO誤差で安定度を比較します。
  • 同時刻画像では、表示ステータスを「基礎値 + 将星補正 + 練達補正」に分解し、次将星や次練達の伸び方も追跡しています。
  • この比較は、手元の画像サンプルを増やすほど強くなります。今は「魅力が効くか」「天賦が隠れ要素の代理か」「将星と練達がどこまで直接戦力へ効くか」を見る用途向けです。

画像から起こした15件をすぐ呼び出せます。楊端和 / シュンメン / 玄峰 / 尚鹿 / バジオウは将星と練達の分解にも対応しています。

魅力は比較用です。精度が上がらない場合は「独立寄与が弱い」仮説を後押しします。

同時刻画像で見えた投資分解

同じタイミングで撮られた将星・練達付き画像から、現在の表示値を基礎値 / 将星 / 練達に分解して表示します。 将星画面の緑値は「次に増える量」、練達画面は「白値 - 緑値」で現在補正を戻せるかを検証しています。

  • 公式指南書の「部隊編成のポイント / 軍勢編成のポイント」、GameWith の編成・補佐・陣形記事を土台にしています。

  • 現在は powerCurrent / powerPotential / burst20s / sustain40s / siegeDps / ehp / controlUptime / cleanseCoverage / formationFit / metaPrior を別軸で持ち、最後だけ最適化方針ごとの重みで合成します。
  • 結果は固定解ではなく、公開ソースを横断して作ったヒューリスティックです。手持ち・簡易育成入力は反映しますが、厳密な戦力式は公開されていないため推定を含みます。
実測戦力CSVと公開根拠

標準推定は公開データ中心、CSV実測優先は貼り付けた現在値の戦力をそのまま使って高戦力編成を組みます。

見出しは「武将名, 戦力, 攻撃, 防御, 戦威, 策略, 魅力, レベル, 天賦, 秘伝」を認識します。戦力列が欠けていても、4戦闘ステ基準の仮説群に Lv / 天賦 / 魅力 / 秘伝を必要に応じて足した中から、最も安定した式で推定補完します。1体でも反映すると自動編成は手持ち限定で組みます。
